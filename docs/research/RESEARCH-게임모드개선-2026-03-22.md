## 3. 경마 모드 — 오벌 트랙 리서치 (tech-researcher)

### 기술 비교표

| 기술/접근법 | 장점 | 단점 | 구현 복잡도 | 추천도 |
|---|---|---|---|---|
| **파라메트릭 타원 경로** | 외부 의존 없음, 경로/속도/방향 정확 제어 | 직접 구현 필요 | 중 | ⭐⭐⭐ |
| **pixi-tween TweenPath** | `drawEllipse()` API로 간단 | 추가 라이브러리, GSAP 충돌 가능 | 하 | ⭐⭐ |
| **GSAP MotionPath Plugin** | 선언적, 기존 GSAP 활용 | SVG 기반이라 PixiJS 직접 적용 어려움 | 상 | ⭐ |
| **pixi-viewport follow** | 카메라 추적 즉시 사용 가능 | 트랙 전체 렌더 필요 → 성능 부담 | 중 | ⭐⭐ |
| **MotionBlurFilter** | 속도감 연출 간단 | kernelSize 큰 경우 성능 저하 | 하 | ⭐⭐⭐ |

### 추천 구현 방식: 파라메트릭 타원 + MotionBlurFilter

#### 핵심 수식

```typescript
// 오벌 트랙 파라미터 (390×844 세로형 화면)
const cx = 195, cy = 420; // 타원 중심
const rx = 150;            // 가로 반경
const ry = 200;            // 세로 반경

// 매 프레임: θ를 속도만큼 증가
horse.theta += horse.speed * delta;

// 위치 업데이트
horse.x = cx + rx * Math.cos(horse.theta);
horse.y = cy + ry * Math.sin(horse.theta);

// 진행 방향에 맞게 회전
horse.rotation = Math.atan2(
  ry * Math.cos(horse.theta),   // dy/dθ
  -rx * Math.sin(horse.theta)   // dx/dθ
);
```

#### 트랙 렌더링 (레이어 구조)

```
Layer 4: 레인 구분선 (arc)
Layer 3: 트랙 표면 (황토색 타원)
Layer 2: 잔디 내부 (초록 타원)
Layer 1: 관중석 (진한 회색 타원)
```

#### 속도감 연출

```typescript
import { MotionBlurFilter } from '@pixi/filter-motion-blur';

const blurFilter = new MotionBlurFilter({ kernelSize: 7 });
horse.container.filters = [blurFilter];

// 매 프레임 속도에 따라 blur 업데이트
const vx = -rx * Math.sin(theta) * speed;
const vy = ry * Math.cos(theta) * speed;
blurFilter.velocity = { x: vx * 0.5, y: vy * 0.5 };
```

#### 카메라 결론

트랙 전체가 390×844 화면에 맞게 설계 → pixi-viewport 불필요. 오벌 트랙 안에서 말들이 빠르게 회전하는 것만으로 충분한 속도감.

## 4. 구슬 레이스 — 멈춤 버그 분석 (tech-researcher)

### 버그 원인 분석

| 원인 후보 | 가능성 | 근거 |
|---|---|---|
| **램프 `frictionStatic` 기본값 0.5** | **매우 높음 (주원인)** | `tan(16°) ≈ 0.287 < max(0.5, 0.1) = 0.5` → 정지마찰이 중력 분력 초과 |
| 램프 각도 0.28rad(≈16°) 너무 완만 | 높음 (보조원인) | 경사각에 비해 정지마찰이 큼 |
| gravity scale 부족 | 중간 | `y=1.2 * scale=0.001` → 실효 중력 약함 |
| 구슬 크기(8px) 대비 램프 두께(10px) | 중간 | 얇은 body 충돌 감지 정확도 저하 |
| enableSleeping 활성화 | 낮음 | 코드상 기본값 false |

### 핵심 원인 상세

Matter.js 정지마찰 결합 공식: `Math.max(bodyA.frictionStatic, bodyB.frictionStatic)`

- 램프 `frictionStatic`: **기본값 0.5** (코드에서 미지정!)
- 구슬 `frictionStatic`: 0.1
- 결합값: `max(0.5, 0.1) = 0.5`
- 물리 조건: `μs < tan(θ)` 이어야 미끄러짐 → `0.5 > 0.287` → **구슬 정지!**

### 권장 수정 파라미터

**구슬 (Marble)**

```typescript
// 현재값 → 권장값
restitution: 0.4   → 0.5
friction: 0.01     → 0.005
frictionAir: 0.007 → 0.003
frictionStatic: 0.1 → 0.02
```

**램프 (핵심 수정)**

```typescript
const ramp = PhysicsWorld.createWall(rampX, rampY, TRACK.rampWidth, TRACK.rampThick, {
  angle,
  restitution: 0.3,
  friction: 0.01,
  frictionStatic: 0,   // ← 이 줄이 빠져있었음! 기본값 0.5가 주원인
});
```

**램프 레이아웃**

```typescript
rampAngle: 0.28  → 0.35    // 약 20° (tan(20°)=0.364 확보)
rampSpacing: 78  → 90      // 간격 넓혀 구슬 공간 확보
rampThick: 10    → 14      // 충돌 감지 안정성 향상
```

**엔진**

```typescript
this.engine = Matter.Engine.create({
  gravity: { x: 0, y: 1, scale: 0.002 },  // scale 2배
  positionIterations: 10,  // 기본 6 → 경사면 정확도 향상
  velocityIterations: 8,   // 기본 4 → 속도 계산 안정화
});
```

## 5. 사다리타기 — 복잡 구조 + 카오스 설계 (architecture-researcher)

### 구조 개선 비교표

| 패턴/접근법 | 예측 불가능성 | 구현 복잡도 | 추천도 |
|---|---|---|---|
| 행 수 증가 (10 → 25~30) | 중 | 하 | ⭐⭐⭐ |
| 가로줄 밀도 70%+ | 상 | 하 | ⭐⭐⭐ |
| 다중 가로줄 (한 행 2개) | 상 | 중 | ⭐⭐⭐ |
| 긴 가로줄 (2칸 건너뛰기) | 상 | 중 | ⭐⭐⭐ |
| 대각선 가로줄 | 중 | 상 | ⭐⭐ |
| 카오스: 트레이싱 중 가로줄 변형 | 최상 | 상 | ⭐⭐⭐ |
| 워프 포인트 (비인접 열 점프) | 최상 | 중 | ⭐⭐⭐ |

### 핵심 원리: Dense Amidakuji

사다리타기 = **순열 연산자**. 각 가로줄 = 인접 열의 swap(transposition). 랜덤 스왑이 충분히 많으면 인간의 뇌가 경로를 추적 불가.

### 카오스 시스템 설계

#### 이벤트 1: Bridge Shuffle (사다리 셔플)
- **트리거**: 트레이싱 30% 지점
- **내용**: 미통과 구간의 가로줄 일부 재배치
- **연출**: `gsap.to(rung, { x: newX, duration: 0.3, ease: "back.out" })` + 깜빡임
- **공정성**: 이미 지나간 구간은 변경 불가

#### 이벤트 2: Warp Zone (워프 포인트)
- **트리거**: 특정 행의 워프 마커 통과 시
- **내용**: 비인접 열로 순간이동
- **연출**: `gsap.fromTo(trace, { scale: 1.5, opacity: 0 }, { scale: 1, opacity: 1 })` + 포털 파티클
- **공정성**: 결과 역산하여 워프 목적지 배치

#### 이벤트 3: Bridge Burst (교차 폭발)
- **트리거**: "폭탄" 마커 통과 시
- **내용**: 해당 행 전체 가로줄 소멸 후 재생성
- **연출**: ShakeEffect + `scaleX: 0→1` 애니메이션

#### 이벤트 4: Climax Approach (슬로우 피니시)
- **트리거**: 마지막 3행 진입
- **내용**: 트레이싱 속도 0.3x 슬로우모션
- **연출**: 기존 `SlowMotionEffect` 재활용

### 추천 최종 구조

```
세로열: 플레이어 수 (2~10)
행 수: 25행 (기존 10행 → 2.5배)
가로줄 밀도: 70% (기존 42%)
다중 가로줄: 6열 이상 시 한 행에 최대 2개 (인접 불가)
긴 가로줄: 20% 확률로 2칸 건너뛰기 (별도 색상 표시)

카오스 이벤트 (1~2회):
- [행 7~9] 워프 포인트 1개
- [행 17~19] Bridge Shuffle 또는 Bridge Burst 중 랜덤 1개

시각 연출:
- 기본 → 워프 순간이동 → 후반 가속 → 마지막 3행 슬로우
- 긴 가로줄: 점선 또는 다른 색상
- 카오스 전 0.5초 경고 이펙트 (화면 진동)
```

| **드롭 게이트 (Gate)** | 직사각형 Body + `isStatic` 토글 | 상 | 하 | ⭐⭐⭐ |

### 단일 Goal 구조 (핵심 변경)

**기존**: 등수별 N개 슬롯 → **변경**: 하단 깔때기형 단일 구멍

```
┌─────────────────────────┐
│    ● ● ● (공 발사)      │  ← 상단
│   핀 배열 (그리드)       │
│  [범퍼][슬링샷][회전판]  │  ← 중간: 장치 구간
│  ┌──┐        ┌──┐       │
│  │  └────────┘  │       │  ← 깔때기 벽 (경사 Body)
│         ▼               │
│       [ 골 ]            │  ← 단일 구멍 (공 1개씩 통과)
└─────────────────────────┘
```

- **골 구멍 크기**: 공 지름의 1.3~1.5배 → 한 번에 1개만 통과
- **도착 순서 = 등수**: rankings 배열에 도착 순서대로 push
- **뽑기 모드**: `'first'` → 첫 도착이 1등 / `'last'` → 마지막 도착이 꼴등

### 다중 공 시스템

- **선택지**: 플레이어당 1개 / 2개 / 3개 (MainMenuScene에서 선택)
- **총 공 수**: 플레이어 수 × 선택 개수 (최대 30개)
- **판정 기준**: 첫 번째 공 도착 기준 (가장 직관적)
  - 1등: 가장 먼저 공이 골에 도착한 플레이어
  - 꼴등: 가장 늦게 공이 골에 도착한 플레이어

### 물리적 고려사항

```typescript
// 공 간 충돌 활성화 (밀어내기 → 예측불가)
// 속도 하한선 (멈춤 방지)
Events.on(engine, 'beforeUpdate', () => {
  balls.forEach(ball => {
    const speed = Vector.magnitude(ball.velocity);
    if (speed < MIN_SPEED && speed > 0) {
      Body.setVelocity(ball, Vector.mult(
        Vector.normalise(ball.velocity), MIN_SPEED
      ));
    }
  });
});

// 골 구멍 위 약한 인력 자석 → 공 자연 수렴 (게임 지연 방지)
```

### 관람형 게임 UX 5대 원칙

1. **예측 불가능성**: 랜덤 luck factor → 순위 변동이 핵심 흥미
2. **러버밴드 메커니즘**: 뒤처진 플레이어에게 카오스 우선 발동 → 역전 드라마
3. **정보 은닉**: 순위 일부만 보여주다 클라이맥스에 전체 공개
4. **사건-절정-결말 아크**: 30초 내 기대→카오스→해소 3막 구조
5. **응원자 전환**: 자신이 응원하는 대상이 있으면 몰입도 폭발

### 뽑기런 차별화 포인트

| 포인트 | 경쟁 제품 대비 차이 |
|---|---|
| 4종 게임 모드 | 대부분 단일 모드 |
| PickMode 이분법 | 1등만 뽑는 구조 탈피 → 꼴등 뽑기 + 벌칙 연출 |
| 30초 극적 아크 | 결과만 보여주는 경쟁 제품 대비 완전한 감정 여정 |

| 모임 최적화 | 2~10명 이름 입력, 집단 관람 경험 |

## 8. 라이브러리 최신 정보 (Context7)

### PixiJS v8
- `Graphics` — arc, ellipse, bezier curve, polygon 모두 지원
- Container `pivot` + `rotation`으로 카메라 회전/추적 구현 가능
- v8에서 Container만 children 가질 수 있음 (`sprite.addChild()` 불가)
- `pixi-viewport` — 2D 카메라 라이브러리 (드래그/줌/스냅), v8 호환

### Matter.js
- `sleepThreshold` — body가 일정 시간 속도 0이면 sleep 진입 (기본 false)
- `frictionStatic` 기본값 **0.5** — 결합 시 `Math.max()` 사용 (주의!)
- `friction` 결합 시 `Math.min()` 사용
- `restitution` — 충돌 시 두 body 중 큰 값 적용

### GSAP
- `MotionPathPlugin` — 곡선 경로 따라 오브젝트 이동 (SVG path 기반)
- `PixiPlugin` — PixiJS 오브젝트 직접 트윈, degrees 단위 회전 지원
- `stagger` — 순차 애니메이션 (사다리 트레이싱에 활용)
