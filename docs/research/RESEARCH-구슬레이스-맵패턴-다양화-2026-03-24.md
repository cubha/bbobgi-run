> 생성일: 2026-03-24
> 프로젝트: bbobgi-run (뽑기런)
> 기반 자료: 없음 (신규 리서치)

## 1. 리서치 배경

현재 `MarbleRaceScene.ts`의 트랙 구성은 **20개 동일한 지그재그 램프 + 보틀넥 2개 + 4×4 핀존 1개**로, 전체 레이스가 단조롭다는 피드백이 있음.
이 리서치는 유사 게임/물리 실제 사례를 분석해 **2D Matter.js 환경에서 즉시 구현 가능한 개선 패턴**을 도출하는 것이 목표.

### 현재 트랙 스펙 요약

| 구성 요소 | 현황 | 문제점 |
|---|---|---|
| 지그재그 램프 | 20개 × 동일 각도(0.35rad) × 동일 폭(220px) | 반복 = 지루함 |
| 보틀넥 | 2개 × 동일 형태(passWidth=90) | 개성 없음 |
| 핀존 | 4×4 균등 격자 1개 | 너무 작고 단순 |
| 분기/다중경로 | 없음 | 매 레이스 동일 결과 |
| 속도 변화 구간 | 없음 | 템포 단조로움 |
| 피날레 구간 | y=1645~2300: 램프 3개뿐 | **655px 데드존** |

## 2. 조사 결과

### 2-1. 유사 게임 분석

#### Jelle's Marble Runs (YouTube 스펙테이터 레이스)
- **Sand Rally**: 자연 지형(나뭇가지, 돌)이 랜덤 장애물 → 동일 트랙도 매번 다른 결과
- **Marbula One**: F1 모사 — 코너 → 스트레이트 → 코너 단순 반복이 아닌 각 섹션에 **다른 특성** 부여
- **핵심 철학**: 물리 법칙만으로 결과가 결정되지만, 장애물 다양성이 매 레이스를 다르게 만듦

#### Marbles on Stream (Steam/Twitch)
- **Narrower**: 트랙 폭을 좁히는 전용 피스 4가지 variant. 병목 역할 + "점프 전 마블 1열 정렬"
- **Splitter**: Y자 분기. 마블을 무작위로 좌/우 경로로 분리
- **Speed Boost**: 일정 구간에서 강제 가속
- **Moving Pins**: 회전/왕복 운동하는 핀 (정적 핀과 구분)
- **공식 가이드라인**: "데드스톱(마블이 완전히 멈추는 현상)은 지루한 레이스를 만드는 최악의 실수"

#### Simple Marble Race (모바일)
- 80개 이상 프리빌트 트랙: **Loops → Drops → Zigzag → Funnel → Hazard** 섹션이 명확하게 구분됨
- 각 레이스마다 다른 섹션 조합 사용 → 반복감 없음

#### Marble Race Creator (Ugly Pixels / itch.io)
- Rectangular walls + Circular walls + Punch bags + Splitter module 등 이질적 조합
- spinning gates, bumpers, ramps, funnels, loops, drops, chicanes, rough terrain
- 커뮤니티 피드백 1위: "결과를 예측할 수 없도록 장애물 배치 다양화"

#### Fall Guys (관람형 레이스 레벨 디자인 모범 사례)
- **레이어드 난이도**: 초반 쉬움 → 결승선 근처 어려움
- **초크포인트 원칙**: "스킬 있는 플레이어가 너무 쉽게 도망가지 않도록" 좁은 구간 의도적 배치
- **Slime Climb**: 위로 올라갈수록 난이도 증가 → 공간과 난이도의 직관적 연계
- **비예측성 유지**: 장애물이 완전히 무작위여야 구경이 재밌음

#### 국내 뽑기/추첨 구슬 게임
- planck.js / matter.js 기반 구현이 주류
- 핀볼 스타일 핀 배열 + 하단 골 구멍 구조
- 장애물보다 **랜덤성**에 집중 (추첨 공정성 우선)

### 2-2. 섹션 아키타입 분류

성공적인 마블 레이스 트랙은 다음 5가지 섹션 아키타입의 조합으로 구성된다.

| 섹션 타입 | 역할 | 특징 |
|---|---|---|
| **스피드 존** | 가속, 관람 흥미 일시 하락 | 넓은 폭, 완만한 경사 |
| **컨트롤 존** | 예측 불가 결과 생성 | 핀, 스플리터, 범퍼 |
| **초크포인트** | 순위 압축/재편 | 좁은 통로, 펀넬 |
| **카오스 존** | 역전 기회 창출 | 장애물 밀집, 랜덤성 극대화 |
| **피날레** | 긴장감 클라이맥스 | 좁아지는 통로 + 직선 결승 |

### 2-3. 장애물별 게임플레이 효과

| 장애물 | 물리 동작 | 관전 포인트 | 2D 구현 난이도 |
|---|---|---|---|
| 지그재그 경사로 | 방향 전환, 속도 유지 | 앞서거니 뒤서거니 | ★ (현재 있음) |
| 깔때기/병목 | 좁아지며 속도 증가 | 경쟁 집중 | ★ (현재 있음) |
| 핀 배열 (staggered) | 충돌 후 불규칙 산란 | 역전 가능성 | ★ (현재 있음) |
| 계단식 드롭 | 단계별 낙하 | 순위 가시화 | ★★ |
| 분기 경로 (fork) | 랜덤 방향 선택 | "어디로 갈까?" 긴장감 | ★★★ |
| 나선형 구간 | 원심력, 점차 가속 | 예측 불가 출구 | ★★ |
| 스피너 | 회전 충격량 | 극적 변동 | ★★ |
| 점프대 | 포물선 → 불규칙 착지 | 흥분 피크 | ★★ |
| 터널 (시각) | 구슬 일시 가림 | 신비감 | ★ (시각만) |
| 비대칭 보틀넥 | 한쪽이 더 좁음 | 클러스터링 | ★ |
| 컨베이어 벨트 | 일방향 강제 이동 | 의도된 혼돈 | ★★ |

## 3. 현재 구현 문제점 심층 분석

### 3-1. 구조적 단조로움 (Critical)

```
현재 트랙 구성 (2400px):
[160~1940px] 지그재그 20개 × 90px 간격 = 1800px (75%) 동일 패턴
[655px]      병목 1
[1080px]     핀존 (4×4)
[1645px]     병목 2
[2300px]     골
```

전체 2400px 중 **1800px(75%)가 동일한 지그재그** → 10초 이후 "아, 또 같은 구간" 인지.
관련 원칙 위반: Marbles on Stream 가이드 "단순히 피스를 이어붙이는 것은 피하라"

### 3-2. 피날레 데드존 (Critical — 가장 시급)

**y=1645(병목2) ~ y=2300(골) = 655px 구간에 장애물 사실상 없음**
- 램프 ramp17~19만 존재 (단순 지그재그 3개)
- 이 구간에서 순위가 그대로 굳어버림
- 마지막 역전 기회가 없음 → 결과가 결승 전 이미 결정됨

### 3-3. 핀존 크기 부족 (Moderate)

현재 4×4 = 16핀, 간격 50px → 실제 커버 폭 = `(4-1) × 50 = 150px`
트랙 폭 330px 대비 **45%만 커버** → 양쪽 90px 구간은 핀 영향 없이 직선 낙하

### 3-4. 병목 배치 불균형 (Moderate)

- 병목1: y=655 (전체 27%)
- 병목2: y=1645 (전체 68%)
- 두 병목 사이 990px 구간에 역전 메커니즘 = 핀존 하나뿐
- 병목2 이후 결승까지 아무런 장애물 없음 (데드존)

### 3-5. 속도 균일성 (Minor)

중력 고정(`{ y: 1.2 }`) + 동일 램프 각도 → 구슬들이 비슷한 속도로 함께 내려감 → 순위 변동 드라마 없음

## 4. 게임 디자인 원칙

### 4-1. 페이싱(Pacing) 강도 곡선

```
강도
높음 ─────────────────────────────────────
       /\         /\         /\       /\
      /  \   /\  /  \   /\  /  \    /  \
     /    \ /  \/    \ /  \/    \  /    \
낮음──────────────────────────────────────
     [출발] [A]  [B]   [C]  [D]  [E]  [골]
```

- 고강도 구간(핀존·병목)과 저강도 구간(스트레이트·완만 램프)이 반드시 교번
- 30초 게임 기준: 고강도 4~5초 / 저강도 2~3초 사이클

### 4-2. 역전 메커니즘 황금 비율

| 유형 | 수단 | 비율 |
|---|---|---|
| **수동적 역전** | 트랙 구조(핀존·분기·병목) | 70% |
| **능동적 역전** | 이벤트(꼴찌 부스터·선두 낙뢰) | 30% |

현재 구현: 수동 역전 수단이 부족(핀존 1개, 동일 병목 2개), 능동 역전에 의존도 과다.

### 4-3. 데드스톱 방지 원칙

모든 통로 폭 ≥ 마블 지름(16px) × 3 = 48px 이상 확보 필수.
현재 보틀넥 passWidth=90px → OK, 핀 간격 50px → OK.

## 5. 권장안

### ★★★ 1순위: 5-Zone 트랙 구조 재설계

```
[Zone A: 출발 산개] y=100~520 (420px)
  ├ 와이드 램프 4개: width=240px, angle=0.20
  └ 목표: 마블 고르게 산개, 초기 위치 형성

[Zone B: 첫 번째 압축] y=520~800 (280px)
  ├ 병목 1 (y=655, passWidth=80, 비대칭 ±0.3)
  ├ 점진적 가이드레일 (280→80px 수렴)
  └ 목표: 첫 번째 드라마틱 역전

[Zone C: 핀볼 카오스] y=800~1300 (500px)
  ├ 확장 핀존: 7×5 다이아몬드 배열, spacing=40px (35핀)
  ├ 핀존 전폭 커버: 240px/330px = 73%
  ├ 중간 지그재그 3개: width=200px, angle=0.3
  └ 목표: 순위 산란·역전 극대화

[Zone D: 분기+카오스] y=1300~1800 (500px)
  ├ 스팁 램프 3개: width=180px, angle=0.45
  ├ 병목 2 (y=1645, passWidth=60, 더 좁음, ±0.5)
  ├ 계단식 드롭 4개 선반 (y=1700~1780)
  └ 목표: 클라이맥스 긴장감

[Zone E: 피날레] y=1800~2300 (500px)  ← 현재 데드존 해결
  ├ 좁아지는 가이드레일 (180→120px)
  ├ 마지막 핀존 소형 (5×3, 200px 커버)
  ├ 스팁 경사 2개 (angle=0.5, width=160px)
  ├ 짧은 직선 + 결승선 (y=2300)
  └ 목표: 마지막 역전 기회 + 긴장감 최고조
```

### ★★★ 2순위: 계단식 드롭 섹션 (staircase drop)

Zone D에 구현. 지그재그 대비 시각적으로 완전히 다른 패턴.

```typescript
// buildStaircase(startY, steps=4) 신규 함수
for (let i = 0; i < steps; i++) {
  const dir = i % 2 === 0 ? 1 : -1;
  const shelfX = cx + dir * (trackW * 0.22);  // 좌우 편심
  const shelfY = startY + i * 65;
  const shelf = PhysicsWorld.createWall(shelfX, shelfY, 130, 10, {
    angle: dir * 0.05,   // 약한 기울기로 자연 흘림
    restitution: 0.25,
    friction: 0.04,
  });
}
```
- 각 선반이 좁고(130px) 짧아서 구슬이 빠르게 다음 선반으로 낙하
- 순위 변동 가능성 높음 (낙하 지점 불규칙)

### ★★★ 3순위: 핀존 7×5 다이아몬드 확대

```typescript
// 현재: rows=4, cols=4, spacing=50 → 커버 폭 150px
// 개선: rows=5, cols=7, spacing=40 → 커버 폭 240px (73%)
const cols = 7, rows = 5;
const pinSpacing = 40;  // 50 → 40
// 기존 buildPinZone 로직 그대로, 수치만 변경
```

### ★★ 4순위: 비대칭 보틀넥

병목1: 통로 중심 `cx → cx-25` 비대칭 이동.
- 왼쪽 구슬 유리 → 자연스러운 클러스터링
- 병목1과 병목2가 같은 형태에서 벗어남

### ★ 5순위: 분기 경로 (fork)

```
Y자 분기점 (y=1350):
├ 좌측 경로 (폭 120px): 완만한 S자 3~4개 램프 (느리지만 안전)
└ 우측 경로 (폭 120px): 급경사 + 소형 핀 3개 (빠르지만 위험)
y=1700 재합류 → 역전 포인트
```
구현 복잡도 높음 — 경로 합류 로직·카메라 처리 필요. 최후순위.
