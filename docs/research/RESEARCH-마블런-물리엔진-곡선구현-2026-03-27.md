# 마블런 물리엔진 & 곡선 구현 리서치 보고서

> 조사일: 2026-03-27 | 에이전트: tech-researcher + arch-researcher + market-researcher + Context7

## 1. 핵심 결론

### 권장안: Planck.js 마이그레이션 (1순위)

**Matter.js의 근본 한계:**
- ChainShape 없음 → 곡선 레일을 수십 개 직사각형으로 근사해야 함
- Kinematic Body 없음 → `setAngle()` 수동 호출 우회만 가능
- Joint 시스템 없음 → 시소/물레방아/기어 구현 불가
- Ghost Collision 자동 처리 없음 → 세그먼트 이음새에서 구슬 걸림

**Planck.js가 해결하는 것:**
- `ChainShape(vertices, loop?)` → 곡선 지형을 꼭짓점 배열로 직접 생성, ghost collision 자동 처리
- `'kinematic'` body type → 물레방아/풍차 정확한 물리
- `RevoluteJoint` → 시소/회전축, `PrismaticJoint` → 엘리베이터
- Box2D 기반 → 검증된 20년 역사의 물리 알고리즘
- 순수 TypeScript → WASM 없이 동기 로드, 기존 Ticker 루프와 충돌 없음

**마이그레이션 난이도: 보통 (2~3일)**
- `PhysicsWorld.ts` 래퍼 내부만 교체, 외부 인터페이스 유지
- `Marble.ts` sync 로직 소폭 수정
- PixiJS 렌더링 코드 무변경

## 3. Planck.js 핵심 API — 마블런 구현

### 3-1. ChainShape (곡선 레일)

```typescript
import { World, Vec2, Chain, Circle } from 'planck';

// 반원 U턴 레일 (16분할)
const arcPoints: Vec2[] = [];
for (let i = 0; i <= 16; i++) {
  const angle = Math.PI * (i / 16);
  arcPoints.push(Vec2(Math.cos(angle) * 120, Math.sin(angle) * 120));
}
const outerChain = new Chain(arcPoints, false); // open chain
groundBody.createFixture(outerChain, { friction: 0.3 });

// 나선 하강로 — 점 배열만 정의하면 됨
const spiralPoints = generateSpiralPoints(radius, turns, drop);
const spiralChain = new Chain(spiralPoints, false);
```

### 3-2. RevoluteJoint (물레방아/시소)

```typescript
// 물레방아: kinematic body + angular velocity
const wheel = world.createBody({ type: 'kinematic', position: Vec2(cx, cy) });
wheel.createFixture(new Box(bladeLen/2, bladeThick/2));
wheel.setAngularVelocity(Math.PI); // 초당 π 라디안

// 시소: dynamic body + revolute joint
const beam = world.createBody({ type: 'dynamic', position: Vec2(cx, cy) });
beam.createFixture(new Box(beamW/2, beamH/2), { density: 0.005 });
const pivot = world.createBody({ type: 'static', position: Vec2(cx, cy) });
world.createJoint(RevoluteJoint({}, pivot, beam, Vec2(cx, cy)));
```

### 3-3. 닫힌 루프 (360° 채널)

```typescript
// loop=true로 닫힌 곡선 생성
const loopPoints = generateCirclePoints(radius, 24);
const loopChain = new Chain(loopPoints, true); // closed loop!
```

## 4. Matter.js → Planck.js 마이그레이션 매핑

| Matter.js | Planck.js | 비고 |
|---|---|---|
| `Engine.create()` | `World({ gravity: Vec2(0, 9.8) })` | y축 반전 주의 |
| `Bodies.circle(x,y,r)` | `body.createFixture(new Circle(r))` | 바디+픽스처 분리 |
| `Bodies.rectangle(x,y,w,h)` | `body.createFixture(new Box(w/2,h/2))` | half-size |
| `body.position.x` | `body.getPosition().x` | getter 메서드 |
| `Body.setVelocity(b,v)` | `body.setLinearVelocity(v)` | |
| `Body.setAngle(b,a)` | `body.setAngle(a)` | |
| `Body.applyForce(b,p,f)` | `body.applyForce(f, p)` | 인자 순서 변경 |
| `Events.on(engine,'collisionStart')` | `world.on('begin-contact', cb)` | |
| `Composite.add(world, body)` | `world.createBody()` | 팩토리 패턴 |
| 없음 | `new Chain(vertices)` | **신규** |
| 없음 | `RevoluteJoint({}, a, b, anchor)` | **신규** |
| 없음 | `body.type = 'kinematic'` | **신규** |

## 6. 곡선 구현 패턴 비교

| 패턴 | 곡선정밀도 | 구현난이도 | Matter.js | Planck.js | Ghost문제 |
|---|---|---|---|---|---|
| **ChainShape** | 높음 | 중 | ❌ | ✅ 네이티브 | 자동제거 |
| **다각형 근사** | 중~높음 | 낮음 | ✅ | ✅ | 수동처리 |
| **센서+힘 가이드** | 낮음 | 높음 | ✅ | ✅ | 없음 |
| **Kinematic setAngle** | N/A | 낮음 | ✅ 수동 | ✅ 네이티브 | N/A |

### Planck.js 전환 시 구조물별 권장 패턴

| 구조물 | 패턴 | 코드 |
|---|---|---|
| U턴, 나선, 루프 | ChainShape | `new Chain(arcVertices)` |
| 직선 채널/램프 | Box + Edge | `new Box(w/2, h/2)` |
| 물레방아/풍차 | Kinematic Body | `{ type: 'kinematic' }` |
| 시소 | Dynamic + RevoluteJoint | `RevoluteJoint({}, pivot, beam)` |
| 트램폴린 | 높은 restitution Box | `{ restitution: 1.5 }` |
| 밀폐 채널 | 이중 ChainShape (내벽+외벽) | 2개 Chain |

## 7. 결론 및 제안

### Option A: Planck.js 전환 (권장)
- **장점**: ChainShape로 곡선 문제 근본 해결, Joint로 물레방아/시소 정확 구현
- **비용**: PhysicsWorld.ts 재작성 2~3일
- **리스크**: npm 다운로드 수 낮음 (610/주), 하지만 코드 품질은 높음 (TS 99%, 벤치마크 95.1)

### Option B: Matter.js 유지 + 다각형 근사 강화
- **장점**: 마이그레이션 비용 0
- **비용**: 세그먼트별 수십 개 벽 수동 배치, ghost collision 수동 처리
- **리스크**: 곡선 품질 한계, 물레방아/시소 정확도 부족

### Option C: Rapier2D 전환 (대규모 확장 시)
- **장점**: WASM 최고 성능, Polyline+Heightfield
- **비용**: 번들 6.45MB, API 패러다임 전환 4~6일
- **리스크**: 현재 규모(10~20구슬)에서는 과도한 투자

### 최종 권장: **Option A (Planck.js)**

CLAUDE.md에 이미 "Planck.js는 리스크 시에만"으로 언급된 후보이며, 현재 상황이 정확히 그 "리스크 시"에 해당한다. Matter.js의 근본 한계(ChainShape/Joint 없음)로 마블런 핵심 구조물 구현이 불가능한 상태.
