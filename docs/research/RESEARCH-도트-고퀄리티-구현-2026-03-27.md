# 고퀄리티 도트/픽셀아트 마블런 구현 리서치

> 조사일: 2026-03-27 | 에이전트: pixel-tech + pixel-ux + Context7

## 1. 핵심 결론 — 즉시 적용 가능한 3줄 설정

```typescript
// 1. 전역 텍스처 Nearest Neighbor (픽셀 선명도)
TextureStyle.defaultOptions.scaleMode = 'nearest';

// 2. 서브픽셀 방지 (정수 좌표 강제)
await app.init({ roundPixels: true });

// 3. CSS 레벨 업스케일
app.canvas.style.imageRendering = 'pixelated';
```

이 3줄만으로 현재 Graphics API 드로잉도 즉시 "도트 느낌"이 강화된다.

## 2. 단계별 업그레이드 로드맵

### Phase A: 즉시 (코드 수정만, 에셋 불필요)

| 항목 | 방법 | 공수 |
|---|---|---|
| 픽셀 렌더링 설정 | 위 3줄 코드 | 5분 |
| `pixi-filters` 설치 | `npm i pixi-filters` | 5분 |
| 구슬 GlowFilter | 1등 구슬에 `GlowFilter({ outerStrength: 2, color: gold })` | 30분 |
| 카오스 GlitchFilter | `ChaosEffect.ts`에 `GlitchFilter` 추가 | 30분 |
| 포토피니시 색수차 | `RGBSplitFilter` ± 2px | 30분 |
| 구슬 트레일(잔상) | 이전 3~5프레임 위치에 alpha 감쇠 렌더 | 1시간 |

### Phase B: 중기 (스프라이트 에셋 제작)

| 항목 | 방법 | 공수 |
|---|---|---|
| 구슬 도트 스프라이트 | Piskel/LibreSprite로 24x24 PNG 제작 → Sprite 교체 | 2~3시간 |
| 구슬 굴림 애니메이션 | AnimatedSprite 4~8프레임 | 2~3시간 |
| 도트 파티클 | 4x4 픽셀 블록 PNG + ParticleContainer | 2시간 |
| 트랙 타일 텍스처 | 세그먼트별 도트 텍스처 (하이라이트 엣지) | 3~4시간 |
| 팔레트 확장 | PICO-8 16색 → DawnBringer 32색 | 2시간 |

### Phase C: 고급 (선택사항)

| 항목 | 방법 | 공수 |
|---|---|---|
| CRT 스캔라인 | `CRTFilter` 전체 화면 | 1시간 |
| 노멀맵 라이팅 | `pixijs-userland/lights` 플러그인 | 1~2일 |
| 배경 패럴랙스 | 3레이어 배경 (원경 0.2x, 중경 0.5x, 근경 1x) | 3~4시간 |

## 3. PixiJS v8 픽셀아트 렌더링 옵션

| 방식 | 품질 | 성능 | 구현 난이도 | 추천도 |
|---|---|---|---|---|
| **Graphics API** (현재) | 보통 (안티앨리어싱) | 보통 | 낮음 | 현재 |
| **Sprite (PNG)** | 최고 (도트 원본) | 높음 (GPU 배치) | 보통 | **최고 추천** |
| **AnimatedSprite** | 최고 | 높음 | 보통 | 추천 (구슬 굴림) |
| **RenderTexture 스케일업** | 높음 (풀 레트로) | 중간 | 높음 | 선택 |
| **ParticleContainer** | 높음 | 최고 (100만 파티클) | 낮음 | 이펙트 전용 |

### Sprite 교체 패턴 (Marble.ts)

```typescript
// 현재: Graphics API 드로잉
private bodyGfx: Graphics;

// 변경: 도트 스프라이트
private bodySprite: Sprite; // or AnimatedSprite

sync(): void {
  this.container.x = Math.round(this.body.position.x);
  this.container.y = Math.round(this.body.position.y);
  this.bodySprite.rotation = this.body.angle;
}
```

## 4. pixi-filters 활용 (v6.x — PixiJS v8 호환)

| 필터 | 효과 | 마블런 활용 | 성능 |
|---|---|---|---|
| **GlowFilter** | 외곽 글로우 | 1등 구슬 강조, 순위 변동 | 낮음 |
| **AdvancedBloomFilter** | 빛번짐 | 네온 배경, 결승선 | 중간 |
| **OutlineFilter** | 외곽선 | 모든 구슬 윤곽 강조 | 낮음 |
| **GlitchFilter** | 글리치 슬라이스 | 카오스 이벤트 진입 | 낮음 |
| **ShockwaveFilter** | 충격파 | 폭발 이벤트 지점 | 낮음 |
| **CRTFilter** | 스캔라인+비네팅 | 전체 화면 레트로 | 중간 |
| **RGBSplitFilter** | 색수차 | 포토피니시 긴장감 | 최소 |
| **PixelateFilter** | 픽셀 블록화 | 슬로모 진입 연출 | 낮음 |
| **ColorMapFilter** | LUT 팔레트 매핑 | PICO-8 색상 제한 | 낮음 |

### 적용 예시 — 카오스 이벤트

```typescript
import { GlitchFilter, ShockwaveFilter } from 'pixi-filters';

// 카오스 이벤트 발생 시
const glitch = new GlitchFilter({ slices: 10, offset: 5 });
this.hudContainer.filters = [glitch];
setTimeout(() => { this.hudContainer.filters = []; }, 500);

// 폭발 지점 충격파
const shock = new ShockwaveFilter([cx, cy], {
  amplitude: 20, wavelength: 60, speed: 300,
});
this.worldContainer.filters = [shock];
```

## 5. 팔레트 업그레이드

### PICO-8 16색 → DawnBringer 32색 (1순위 추천)

현재 16색으로는 10명 플레이어 구슬 색상 구분이 부족. DB32는 네온+자연톤 균형 우수.

```
핵심 네온:   #fbf236(옐로), #99e550(그린), #5fcde4(시안),
             #5b6ee1(블루), #d95763(레드), #d77bba(핑크)
다크:        #000000, #222034, #45283c, #323c39
중간:        #8f563b, #df7126, #6abe30, #37946e
밝음:        #cbdbfc, #9badb7, #847e87
```

### Neon Space 10색 (구슬 전용 서브팔레트)

10명에 1:1 매핑. 모든 색이 명확히 구분됨.

```
#df0772, #fe546f, #ff9e7d, #ffd080, #fffdff
#0bffe6, #01cbcf, #0188a5, #3e3264, #352a55
```

## 7. 마블런 비주얼 연출 제안

### 구슬 개성 부여

```
1. 색상: Neon Space 10색 → 플레이어 인덱스별 자동 배정
2. 글로우: GlowFilter({ outerStrength: 1, color: 플레이어색상 })
3. 트레일: 이전 3~5프레임 alpha 감쇠 (속도 비례 길이)
4. 이름태그: BitmapText 도트폰트 (구슬 위 8px)
```

### 이벤트 연출

```
역전 (3위↑ 변동):  노란 별 파티클 + 순위표 쉐이크 + Bloom 순간 상승
카오스 (20~25초):  GlitchFilter 0.5초 + ShockwaveFilter + 경고 배너
포토피니시:        0.3x 슬로모 + RGBSplit ±2px + 줌인 1.4x + 비네팅
1등 확정:          폭죽 파티클 + 골드 글로우 폭발
꼴등 확정:         전기 파티클 + 구슬 흔들림
```

### 트랙 구조물 렌더링

```
레이어 구조:
  L0: 배경 그라데이션 (Cyberpunk Neons 어두운 계열)
  L1: 원경 장식 (패럴랙스 0.2x)
  L2: 트랙 구조물 (물리 바디 1:1)
  L3: 구슬 (물리 매핑)
  L4: 이펙트 파티클
  L5: HUD (카메라 비추적)

벽면 스타일:
  - 상단 엣지 1px 네온 하이라이트
  - 배경 4px 간격 격자 패턴
  - 체크포인트: 네온 사인 깜빡임 (GSAP alpha 트윈)
```
